软件和软件工程

发表于 2017-03-26 | 分类于 软件工程

要点浏览

  • 完整的软件产品包括：可以在各种不同的规模及体系结构的计算机上运行的程序，程序运行的过程中产生的各种结果，以及各种描述信息，这些信息可以以拷贝或者是各种电子媒介形式存在。
  • 计算机软件是由专业人员开发并长期维护的软件产品 。
  • 软件工程包括过程，各种方法以及各类工具，以协助专业人员构建高质量的计算机软件。

软件的本质

  • 软件是：（1）指令的集合（计算机程序），通过执行这些指令可以满足预期的特征、功能和性能需求（2）数据结构，使得程序可以合理利用信息（3）软件描述信息，它以硬拷贝和虚拟的形式存在，用来描述程序的操作和使用。
  • 软件是逻辑的而非物理的系统元素，因此，软件和硬件具有完全不同的特性：（1）软件是设计开发的，而不是传统意义上生产制作的（2）软件不会“磨损”，但是软件退化的确存在，不断地变更时软件退化的根本原因（3）虽然整个工业向着基于构建的构建模式发展，然而大多数软件仍是根据实际的顾客需求定制的。
  • 今天，计算机软件可分为七大类，系统软件、应用软件、工程/科学软件、嵌入式软件、产品线软件、web应用软件、人工智能软件。

软件工程

  • 在制定软件解决方案之前，必须尽力理解问题；设计已经成为关键活动；软件必须保持高质量；软件需具备可维护性。
  • 软件工程是（1）将系统的，规范的，可量化的方法应用于软件的开发、运行和维护，即将工程化方法应用于软件（2）在（1）中所述方法的研究。
  • 支持软件工程的根基在于质量关注点，软件工程的基础是过程层，软件工程的方法为构建软件提供了技术上的解决办法，软件工程工具为过程和方法提供了自动化或半自动化的支持。

软件过程

  • 一个通用的软件过程框架通常包含以下五个活动：沟通，策划，建模，构造，部署。
  • 过程说明模型强调对过程活动和任务的详细定义、识别和应用，其目的是提高软件质量，项目的可管理性和对于交付时间和项目费用的可预测性，并对软件工程师构造系统所必须的工作提供指导。
  • 敏捷过程模型强调项目的灵活性，并在一些基本原则的指导下，采用非正式的方式执行软件的过程。

软件工程的实践

  • 实践的精髓：理解问题（沟通和分析）、计划解决方案（建模和软件设计）、实施计划（代码生成）、检查结果（测试和质量保证）
  • 一般原则：存在原则、保持简洁、保持愿景、关注使用者、面向未来、计划复用、认真思考
