功能测试技术

发表于 2017-03-28 | 分类于 软件测试

功能测试技术举报这样的潜能：它能够把针对产品特定功能范围内的测试覆盖率逐渐提升到一个很高的置信度，同时降低测试用例的冗余数量。

等价类划分 ECP

在设计ECP测试之前，我们有必要仔细为某个输入和输出参数涉及的变量数据实施解耦和建模，以将其模塑成分离的合法及非法分类子集。然后，ECP测试就可以经由这样的手段导出：创建每个合法分类子集的合集，直到测试取遍合法分类的所有子集为止。然后再对非法数据子集依次评估。

变量分解

为了帮助我们把数据分解在每一个类中的离散子集，《软件测试艺术》中提出了四个启发式方法，包括值的范围、变量相似组、唯一值和特殊值。

  • 范围：在数据的临近集合中最小值和最大值间的任何数据点应产生相同的结果。
  • 组：只要元素被等价的处理，那么元素组是允许的。
  • 唯一值：在类或者子集中的数据可能以不同于同一类或子集中的其他数据的方式被处理。
  • 特殊值：条件必须提供或者必须不被提供。

参数子集分析

对子集的参数进行分析，得到范围更小的子集

等价类划分测试

对所有参数创建合法类子集的组合，直至在验证测试中至少包含一次所有的合法类子集，接着，对于每一个参数单独测试的每一个非法类子集，把其他参数设置成一些合法值。等价类划分的所有测试都不专门针对边界条件，这是因为ECP是用于识别特定类型的错误，而边界值测试是用于识别不同类型的错误。

边界值分析 BVA

历史经验表明，相当多的问题出现在线性变量的边界值上，所以为了防止忽略边界类的缺陷，我们必须仔细分析线性变量的边界条件。当BVA与ECP结合使用，BVA这个功能技术可以有效地分析独立输入和输出参数等线性参数的边界值。BVA或边界测试检测以下类型的错误特别有用：

  • 错误的数据类型的人工约束
  • 错误地分配关系运算符
  • 数据类型的封装
  • 循环结构问题
  • 差一错误

定义边界测试

Paul Jorgensen建议用于基本边界值分析的测试数量可以用4n+1来计算（或者6n+1包括健壮性边界值测试的最小值-1和最大值+1数值），这里的n等于独立参数的个数。

隐性边界

不是所有的边界都能通过数字型输入或输出来确定

边界值分析小结

边界值分析是一个以介于特定边界值上下的数据作为研究目标的功能性技术，对处在边界条件的变量进行系统分析增加测试的有效性，提供更优良的信息，增加检测特定类缺陷的可能，结合3（BC）公式，可以更好的估计所需边界测试的数量，并相比于其他公式，可以提供更广泛的测试集合。因为测试都是基于确定的边界数值，而不是基于参数的数量。边界值分析的有效性取决于我们是否有能力将离散变量分解为等价类，并确定重要的边界条件。

组合分析

组合测试的途径

目前，通常有两种测试参数相互作用的途径。第一种途径一般涉及随机或者特定的方法，第二种途径包括了更加系统的过程，随机评估的方法包括最优猜测或特定测试和随机选择，而系统性评估方法包括测试每个变量或分支（EC），基准测试（BC），正交陈列（OA），组合测试和穷举测试（AC）。

组合分析实践

  • 新建BC测试的输入矩阵
  • 新建关于通用或可能的场景和历史性故障指标的输入矩阵
  • 为每个参数新建可变参数的输入模型文件
  • 定制模型文件来排除互斥的可变状态或固定的参数
  • 将输入文件应用到PICT工具
  • 复审工具的输出信息
  • 如果有必要，提炼和定制模型文件
  • 将输入文件重新导入PICT工具
  • 重新验证输出信息
  • 执行测试
